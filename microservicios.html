¿Qué son los Microservicios?

Un enfoque arquitectónico nativo de la nube

Concepto Central

Los microservicios son una arquitectura en la que una sola aplicación se conforma de muchos componentes más pequeños que están poco acoplados y se pueden desplegar de forma independiente.

  • Tecnología propia: Cada servicio incluye su propia tecnología, abarcando su propia base de datos y modelo de gestión.
  • Comunicación: Se comunican entre sí utilizando APIs REST, transmisión de eventos y agentes de mensajes.
  • Contexto delimitado: Se organizan por capacidades empresariales específicas, estableciendo límites claros entre cada servicio.
Lo que NO son los microservicios

Arquitectura monolítica: A diferencia del modelo monolítico (una aplicación grande y muy acoplada), los microservicios componen una aplicación a partir de muchos servicios pequeños.

SOA (Arquitectura Orientada a Servicios): Mientras que SOA fue un esfuerzo para estandarizar cómo todos los servicios de una empresa se comunican a través de un bus central, los microservicios son específicos de una aplicación.

Beneficios para las Organizaciones

Despliegue Independiente

Permite actualizar el código, agregar nuevas características o solucionar errores sin tocar toda la aplicación, reduciendo las frustraciones de los despliegues monolíticos. También facilita la creación de equipos ágiles y multifuncionales en torno a un solo servicio.

La Herramienta Adecuada

A diferencia del modelo monolítico donde todo comparte una pila, aquí se puede optimizar cada servicio. Por ejemplo, el servicio de pagos podría estar desarrollado en C# conectado a una base de datos robusta como Oracle Database, mientras que el servicio del catálogo de productos podría usar Node.js conectado a MySQL. La tecnología puede evolucionar de manera independiente.

Escalado Preciso

Al implementarse de manera individual, los servicios también se pueden escalar individualmente, requiriendo menos infraestructura al evitar tener que escalar toda la aplicación por la alta demanda de una sola característica.

Herramientas Clave

Plataformas como Kubernetes son esenciales para orquestar y gestionar grandes grupos de contenedores en entornos de microservicios.

Actúan como capa intermedia y proxy inverso, enrutando solicitudes y proporcionando seguridad adicional conforme aumenta el número de servicios.

Para evitar consultas constantes ("encuestas"), las llamadas API se acoplan con sistemas de mensajería (como Apache Kafka) para transmitir cambios de estado en tiempo real.

El Desafío Principal

Los microservicios implican un aumento masivo de complejidad: más servicios, más equipos y despliegues distribuidos. Por lo tanto, dependen directamente del éxito en la adopción de la cultura DevOps para automatizar el ciclo de vida, la supervisión y el despliegue.
