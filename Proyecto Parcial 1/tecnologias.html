Tecnologías de Web APIs y sus Subtemas

El ecosistema de las Web APIs es vasto y existen múltiples tecnologías y protocolos para su implementación, dependiendo de los requerimientos de velocidad, peso de los datos y tipado estricto. Las tecnologías subyacentes son fundamentales para el éxito de la comunicación entre sistemas.

Es el estilo arquitectónico más popular. Utiliza los estándares de la web (HTTP) y se basa en recursos (URLs). Es stateless (sin estado), lo que significa que cada petición del cliente contiene toda la información necesaria para que el servidor la entienda. Generalmente retorna datos en formato JSON, que es ligero y fácil de leer tanto para humanos como para máquinas.

Desarrollado por Facebook, GraphQL es un lenguaje de consulta para APIs. Soluciona el problema de "Over-fetching" (traer datos de más) y "Under-fetching" (traer datos de menos) de REST, permitiendo al cliente solicitar exactamente los campos específicos que necesita en una sola petición.

SOAP (Simple Object Access Protocol) es un protocolo más antiguo, altamente tipado y estricto, que utiliza exclusivamente XML para el intercambio de mensajes. Aunque es más pesado y complejo que REST, sigue siendo muy utilizado en entornos empresariales y financieros debido a sus robustos estándares de seguridad integrados (WS-Security).

Manejo de Contenedores

El manejo de contenedores, popularizado masivamente por herramientas como Docker, ha revolucionado la forma en que empaquetamos y distribuimos software. Un contenedor es una unidad estándar de software que empaqueta el código y todas sus dependencias para que la aplicación se ejecute de forma rápida y confiable de un entorno informático a otro.

A diferencia de las máquinas virtuales (VMs) que virtualizan el hardware y requieren un sistema operativo invitado completo y pesado, los contenedores virtualizan el sistema operativo a nivel del núcleo. Esto los hace extremadamente ligeros y rápidos de iniciar. El "Manejo" de estos contenedores implica la creación de imágenes (a través de un Dockerfile), la publicación en registros (como Docker Hub) y, a nivel empresarial, la orquestación de miles de ellos utilizando plataformas como Kubernetes para asegurar alta disponibilidad y balanceo de carga.
