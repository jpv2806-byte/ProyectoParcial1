Tecnologías de Web APIs y sus Subtemas

El ecosistema de las Web APIs es vasto. Una API (Interfaz de Programación de Aplicaciones) es un conjunto de reglas que definen cómo las aplicaciones o dispositivos pueden conectarse y comunicarse entre sí.

Definido por primera vez en el año 2000 por el Dr. Roy Fielding, REST proporciona una gran flexibilidad, lo que lo ha convertido en el método más común para conectar microservicios. Las API REST se comunican mediante solicitudes HTTP para realizar operaciones CRUD (crear, leer, actualizar y eliminar), utilizando métodos como GET, POST, PUT y DELETE.

Para que una API sea verdaderamente RESTful, debe cumplir con seis principios de diseño:

  • Interfaz uniforme: Todas las solicitudes para el mismo recurso deben verse igual, utilizando identificadores uniformes (URI).
  • Desacoplamiento cliente-servidor: Las aplicaciones de cliente y servidor deben ser completamente independientes.
  • Sin estado (Stateless): Cada solicitud debe incluir toda la información necesaria para procesarla; el servidor no almacena sesiones del cliente.
  • Capacidad de caché: Los recursos deben poder almacenarse en caché para mejorar el rendimiento.
  • Arquitectura en capas: La comunicación puede pasar por varios intermediarios sin que el cliente o servidor lo noten.
  • Código bajo demanda: Opcionalmente, se puede enviar código ejecutable al cliente.

GraphQL: Desarrollado por Facebook, es un lenguaje de consulta que permite al cliente solicitar exactamente los datos que necesita, evitando traer información de más o de menos.

SOAP y XML: SOAP impone un marco estricto a los desarrolladores. Es un protocolo más antiguo y altamente tipado que utiliza exclusivamente XML para el intercambio de mensajes, destacando en entornos empresariales por su alta seguridad.

Seguridad en APIs: JSON Web Token (JWT)

Estructura de un JWT

Un JWT consta de tres partes separadas por puntos (.):

  1. Header (Encabezado): Indica el algoritmo utilizado y el tipo de token.
  2. Payload (Carga útil): Contiene los datos de autenticación (como correos o roles) e información de expiración.
  3. Signature (Firma): Garantiza la integridad del token verificando que no haya sido alterado, combinando el header, payload y una clave secreta.

Debido a que las APIs REST "no tienen estado" (no guardan la sesión del usuario), necesitan un mecanismo para identificar quién realiza las peticiones. Aquí es donde entra JWT (JSON Web Token).

Un JWT es un estándar para la autenticación y el intercambio de información definido por RFC7519. Funciona como una credencial digital segura: una vez que el usuario inicia sesión correctamente, el servidor genera este token y se lo entrega al cliente.

¿Cómo se utiliza en el Frontend?

Para acceder a rutas protegidas (como ver el saldo en un banco), la aplicación frontend debe incluir este token en los encabezados (headers) de su petición HTTP, específicamente en el campo de 'Authorization'. El servidor verificará la firma del token; si es válida y no ha expirado, permitirá el acceso al recurso solicitado.

Manejo de Contenedores

El manejo de contenedores ha revolucionado la distribución de software. Los contenedores son paquetes de software ligeros que incluyen el código de las aplicaciones junto con todas sus dependencias, como versiones concretas de entornos de ejecución y bibliotecas indispensables.

A diferencia de las máquinas virtuales que virtualizan a nivel de hardware y requieren un sistema operativo invitado completo, los contenedores virtualizan a nivel del sistema operativo y comparten el kernel del SO anfitrión. Esto los hace extremadamente ligeros y eficientes en el uso de memoria.

Entre sus principales ventajas destacan:

  • Portabilidad total: Se pueden ejecutar de manera uniforme en cualquier lugar, desde el portátil personal de un desarrollador hasta centros de datos privados o la nube pública.
  • Separación de responsabilidades: Los desarrolladores se centran en la lógica de la aplicación y sus dependencias, mientras que los equipos de operaciones de TI operan el despliegue sin preocuparse por configuraciones específicas de software.
  • Aislamiento de aplicaciones: Proporcionan una vista del sistema operativo (CPU, memoria, almacenamiento y red) lógicamente aislada de las demás aplicaciones.
